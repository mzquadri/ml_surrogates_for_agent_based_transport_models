\chapter{Methodology}\label{chapter:methodology}

This chapter describes the GNN architecture adopted for traffic prediction, the three uncertainty quantification methods evaluated in this thesis, and the metrics used to assess both predictive accuracy and uncertainty quality.

\section{Problem Formulation}

We formulate traffic prediction as a node-level regression task on a directed graph. Given a road network $G = (V, E)$ where nodes represent road segments and edges represent connectivity, we aim to predict the change in traffic volume $\Delta v_i$ for each node $i$ after a policy intervention.

\begin{equation}

\end{equation}
representing the change in traffic volume on segment $i$ after the policy intervention is applied.

\section{Model Architecture}

We use the PointNetTransfGAT architecture developed by Natterer~et~al.~\cite{natterer2025ml}, which combines ideas from PointNet~\cite{qi2017pointnet}, graph transformers~\cite{shi2021masked}, and graph attention networks~\cite{velickovic2018graph} into a single end-to-end model for node-level regression on road networks.

\textbf{Note on architectural variants:} Trial~1 uses a \texttt{Linear(64$\to$1)} final prediction layer, while Trials~2--8 use a \texttt{GATConv(64$\to$1)} final layer. This difference makes Trial~1 results not directly comparable with Trials~2--8, and Trial~1 is excluded from MC Dropout UQ experiments (effective dropout $= 0.0$).

\subsection{Geometric Representation}

A key design choice in the architecture is how it encodes the physical geometry of road segments. Each node carries two 2D positional coordinates: the start point of its corresponding road segment, $\mathbf{pos}[:,0,:] \in \mathbb{R}^{N \times 2}$, and its end point, $\mathbf{pos}[:,1,:] \in \mathbb{R}^{N \times 2}$. The two PointNetConv layers~\cite{qi2017pointnet} exploit these coordinates by computing, for each node $i$ and neighbour $j$, the relative displacement $\mathbf{pos}_j - \mathbf{pos}_i$ as part of the local MLP input. This means the first PointNetConv uses the start-point geometry of each segment relative to its neighbours, while the second uses the end-point geometry. The effect is that the model learns not only the graph-topological neighbourhood of each segment but also the spatial proximity of where segments begin and end in the 2D plane---an inductive bias well-suited to the geometry of real road networks.

In Stage~1, the local MLP receives a $7$-dimensional input: the 5 node features concatenated with the 2D start-position displacement $(\mathbf{pos}_j - \mathbf{pos}_i)[:,0,:]$. In Stage~2, the local MLP receives $514$ dimensions: the 512-dimensional embedding from Stage~1 concatenated with the 2D end-position displacement. The TransformerConv layers~\cite{shi2021masked} apply multi-head attention over the graph edges; the GATConv layers~\cite{velickovic2018graph} aggregate neighbourhood information with learned attention weights.

\textbf{Dropout placement:} Dropout is applied within the local and global MLPs of both PointNetConv layers, and between the TransformerConv layers. All dropout calls are gated by the \texttt{use\_dropout} flag (default: \texttt{False}); the flag must be set to \texttt{True} at inference time to activate MC Dropout uncertainty estimation. The architecture also includes a \texttt{predict\_mode\_stats} branch (a TransformerEncoder-based mode predictor) that was implemented in code but was not activated in any of the eight training trials reported in this thesis.

\subsection{Method 1: MC Dropout}

MC Dropout~\cite{gal2016dropout, gal2016thesis} enables uncertainty estimation by keeping dropout active during inference:

\begin{algorithm}[htbp]
\caption[MC Dropout uncertainty estimation]{MC Dropout uncertainty estimation. Given a trained model with dropout layers, $S$ stochastic forward passes yield a predictive mean and standard deviation per node.}
\label{alg:mc_dropout}
\begin{algorithmic}
\STATE \textbf{Input:} Input features $\mathbf{x}$, trained model $f$, number of samples $S$
\STATE \textbf{Output:} Mean prediction $\mu$, uncertainty $\sigma$
\FOR{$s = 1$ to $S$}
    \STATE $\hat{y}_s \leftarrow f(\mathbf{x})$ with dropout enabled
\ENDFOR
\STATE $\mu \leftarrow \frac{1}{S}\sum_s \hat{y}_s$
\STATE $\sigma \leftarrow \sqrt{\frac{1}{S}\sum_s (\hat{y}_s - \mu)^2}$
\RETURN $\mu$, $\sigma$
\end{algorithmic}
\end{algorithm}

We use $S = 30$ MC samples, balancing computational cost against estimate stability.

\begin{figure}[htbp]
\centering
\includegraphics[width=\textwidth]{figures/fig13_mc_dropout_inference.pdf}
\caption[MC Dropout inference: S=30 stochastic forward passes]{MC Dropout inference procedure for T8. A single road network graph is passed through the GNN with dropout active for $S = 30$ stochastic forward passes, yielding 30 prediction vectors $\hat{y}_1, \ldots, \hat{y}_{30}$. Their per-node mean ($\hat{y}$) and standard deviation ($\sigma$) form the final prediction with uncertainty. \subsection{Method 2: Deep Ensembles}

Ensemble variance measures disagreement across independent training runs of the same model:

\begin{equation}
    \sigma_{\text{ens}} = \sqrt{\frac{1}{M}\sum_{m=1}^{M}(f_m(\mathbf{x}) - \bar{f}(\mathbf{x}))^2}
\end{equation}

where $M = 5$ is the number of independently trained models.

\subsection{Method 3: Combined Uncertainty}

We combine MC Dropout and ensemble uncertainty:
\begin{equation}
    \sigma_{\text{combined}} = \sqrt{\sigma_{\text{MC}}^2 + \sigma_{\text{ens}}^2}
\end{equation}

\subsection{Method 4: Split Conformal Prediction}

Split conformal prediction~\cite{vovk2005algorithmic, angelopoulos2023conformal} provides distribution-free prediction intervals with finite-sample coverage guarantees:

\begin{algorithm}[htbp]
\caption[Split conformal prediction]{Split conformal prediction. A calibration set is used to compute a coverage-valid quantile; the evaluation set receives prediction intervals with guaranteed marginal coverage.}
\label{alg:conformal}
\begin{algorithmic}
\STATE \textbf{Input:} Calibration set $\mathcal{D}_\text{cal} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n$, trained model $f$, coverage level $1 - \alpha$
\STATE \textbf{Output:} Prediction interval $C(\mathbf{x}_\text{new})$ for new inputs
\FOR{$i = 1$ to $n$}
    \STATE $s_i \leftarrow |y_i - f(\mathbf{x}_i)|$ \COMMENT{nonconformity score}
\ENDFOR
\STATE $q \leftarrow \text{Quantile}\left(s_1, \ldots, s_n;\; \lceil(n+1)(1-\alpha)\rceil / n\right)$
\RETURN $C(\mathbf{x}_\text{new}) = \left[f(\mathbf{x}_\text{new}) - q,\; f(\mathbf{x}_\text{new}) + q\right]$
\end{algorithmic}
\end{algorithm}

\subsection{Method 5: Selective Prediction}

Selective prediction uses MC Dropout uncertainty to filter out unreliable predictions, retaining only those the model is most confident about:

\begin{algorithm}[htbp]
\caption[Selective prediction via uncertainty filtering]{Selective prediction. Predictions are ranked by MC Dropout uncertainty and only the most confident fraction is retained.}
\label{alg:selective}
\begin{algorithmic}
\STATE \textbf{Input:} Predictions $\hat{y}_1, \ldots, \hat{y}_n$, uncertainties $\sigma_1, \ldots, \sigma_n$, retention fraction $\tau \in (0, 1]$
\STATE \textbf{Output:} Filtered predictions, MAE on retained set
\STATE Sort predictions by ascending $\sigma$
\STATE Retain top-$\tau$ fraction (i.e., the $\lfloor \tau \cdot n \rfloor$ predictions with lowest $\sigma$)
\STATE Compute $\text{MAE}_\tau = \frac{1}{|\mathcal{R}|}\sum_{i \in \mathcal{R}} |y_i - \hat{y}_i|$ on retained set $\mathcal{R}$
\RETURN filtered predictions, $\text{MAE}_\tau$
\end{algorithmic}
\end{algorithm}

\subsection{Method 6: Post-Hoc Temperature Scaling}

Temperature scaling~\cite{guo2017calibration, laves2020well} is a post-hoc recalibration method that learns a single scalar $T$ to rescale uncertainty estimates toward calibration:

\begin{algorithm}[htbp]
\caption[Post-hoc temperature scaling for uncertainty calibration]{Post-hoc temperature scaling. A single temperature parameter $T$ is optimised on a calibration set to minimise the Expected Calibration Error (ECE).}
\label{alg:temperature}
\begin{algorithmic}
\STATE \textbf{Input:} Calibration set $(y_\text{cal}, \hat{y}_\text{cal}, \sigma_\text{cal})$, evaluation set, candidate temperatures $\mathcal{T}$
\STATE \textbf{Output:} Optimal $T^*$, scaled uncertainties $\sigma_\text{scaled}$
\FOR{each $T \in \mathcal{T}$}
    \STATE $\sigma_\text{scaled} \leftarrow \sigma_\text{raw} \times T$
    \STATE Compute Kuleshov ECE on calibration set using $\sigma_\text{scaled}$
\ENDFOR
\STATE $T^* \leftarrow \arg\min_{T \in \mathcal{T}} \text{ECE}(T)$
\STATE Apply $\sigma_\text{scaled} \leftarrow \sigma_\text{raw} \times T^*$ to evaluation set
\RETURN $T^*$, $\sigma_\text{scaled}$
\end{algorithmic}
\end{algorithm}

\section{Evaluation Metrics}

\subsection{Prediction Quality}

Prediction accuracy is evaluated using three standard regression metrics. The \textbf{coefficient of determination} ($R^2$) measures the proportion of variance in the target explained by the model:
\begin{equation}
    R^2 = 1 - \frac{\sum_i (y_i - \hat{y}_i)^2}{\sum_i (y_i - \bar{y})^2}
\end{equation}
The \textbf{mean absolute error} (MAE) gives the average magnitude of prediction errors in the original units (vehicles per hour):
\begin{equation}
    \text{MAE} = \frac{1}{n}\sum_i |y_i - \hat{y}_i|
\end{equation}
The \textbf{root mean square error} (RMSE) penalises large deviations more heavily and is reported for comparability with prior work:
\begin{equation}
    \text{RMSE} = \sqrt{\frac{1}{n}\sum_i (y_i - \hat{y}_i)^2}
\end{equation}

\subsection{Uncertainty Quality}

Uncertainty quality is assessed with three complementary measures. The \textbf{Spearman rank correlation} ($\rho$) captures the monotonic relationship between the predicted uncertainty $\sigma_i$ and the absolute error $|y_i - \hat{y}_i|$; higher values indicate that uncertainty tracks error more faithfully, and $\rho > 0$ is the minimum bar for a useful uncertainty estimate. The \textbf{Pearson correlation} ($r$) provides a linear analogue that is more sensitive to outliers, offering a cross-check on the rank-based result. Finally, the \textbf{$p$-value} of each correlation test is reported to confirm that the observed association is statistically distinguishable from zero.

A well-calibrated uncertainty estimate should correlate positively with prediction error ($\rho > 0$): where uncertainty is high, actual errors should tend to be larger.
